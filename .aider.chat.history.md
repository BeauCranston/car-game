Using `delta` makes movement and acceleration frame-rate independent.

For example:

```csharp

{
```

```csharp
    public Marker2D Marker { get; }
```

```csharp
    /// Advances one physics tick for this wheel.
    /// </summary>
    /// <param name="dt">Delta time in seconds.</param>

    /// <param name="driveTorque">Torque sent from the drivetrain (0 for front wheels).</param>

    {

        Vector2 wheelForward = (-carBody.GlobalTransform.Y).Rotated(steerAngle).Normalized();

            float groundSpeed = wVel.Dot(wheelForward);
```

```csharp
public partial class Wheel : Marker2D
{
    // Physical / setup parameters – can be overridden per wheel in the inspector
    [Export] public float TireRadius { get; set; } = 0.33f;
    [Export] public float WheelInertia { get; set; } = 3f;
    [Export] public float BrakeTorque { get; set; } = 1500f;

    // Pixels per meter should match the car. Could be exported but often car-wide.
    [Export] public float PixelsPerMeter { get; set; } = 100f;

    // Dynamic state
    public float Omega { get; set; } = 0f;               // rad/s

    // Cached reference to the car (parent RigidBody2D)
    private RigidBody2D _carBody;

    public override void _Ready()
    {
        _carBody = GetParent<RigidBody2D>();
        if (_carBody == null)
            GD.PrintErr($"{nameof(Wheel)} must be a child of a RigidBody2D car.");
    }

    /// <summary>
    /// Called every physics tick by the car controller.
    /// </summary>
    public void UpdatePhysics(float dt, float steerAngle, float driveTorque, float brakeInput, float verticalLoad)
    {
        if (_carBody == null)
            return;

        // 1. Wheel forward direction in world space
        Vector2 wheelForward = (-_carBody.GlobalTransform.Y).Rotated(steerAngle).Normalized();

        // 2. Free‑rolling sync
        if (Mathf.Abs(driveTorque) < 0.0001f && Mathf.Abs(brakeInput) < 0.0001f)
        {
            Vector2 wVel = GetVelocityAtWheel();
            float groundSpeed = wVel.Dot(wheelForward);
            Omega = groundSpeed / TireRadius;
        }

        // 3. Brake locking logic
        float prospectiveAppliedBrakeTorque = 0f;
        bool isWheelLocked = false;

        if (brakeInput > 0f && Mathf.Abs(Omega) > 0.01f)
        {
            float brakeDirection = -Mathf.Sign(Omega);
            prospectiveAppliedBrakeTorque = brakeInput * BrakeTorque * brakeDirection;

            Vector2 preVel = GetVelocityAtWheel();
            var preResult = TirePhysics.CalculateTireForces(preVel, wheelForward, Omega, TireRadius, verticalLoad);
            float preFeedback = preResult.forces.Dot(wheelForward) * TireRadius;

            float netTorque = driveTorque - preFeedback + prospectiveAppliedBrakeTorque;
            float prospectiveDeltaOmega = (netTorque / WheelInertia) * dt;

            if (Mathf.Sign(Omega) != Mathf.Sign(Omega + prospectiveDeltaOmega))
            {
                Omega = 0f;
                isWheelLocked = true;
            }
        }

        // 4. Final tire forces
        Vector2 wheelVelocity = GetVelocityAtWheel();
        var result = TirePhysics.CalculateTireForces(wheelVelocity, wheelForward, Omega, TireRadius, verticalLoad);

        // 5. Apply force to car body at this wheel’s offset
        Vector2 pixelForce = result.forces * PixelsPerMeter;
        Vector2 globalWheelOffset = GlobalPosition - _carBody.GlobalPosition;
        _carBody.ApplyForce(pixelForce, globalWheelOffset);

        // 6. Stay locked if required
        if (isWheelLocked || (brakeInput > 0f && Mathf.Abs(Omega) <= 0.01f))
        {
            Omega = 0f;
            return;
        }

        // 7. Integrate omega
        float fxTire = result.forces.Dot(wheelForward);
        float tireFeedbackTorque = fxTire * TireRadius;
        float totalTorque = driveTorque - tireFeedbackTorque + prospectiveAppliedBrakeTorque;
        float angularAcceleration = totalTorque / WheelInertia;
        Omega += angularAcceleration * dt;
        Omega = Mathf.Clamp(Omega, -300f, 300f);
    }

    private Vector2 GetVelocityAtWheel()
    {
        Vector2 globalOffset = GlobalPosition - _carBody.GlobalPosition;
        Vector2 tangentialVelocity = new Vector2(-globalOffset.Y, globalOffset.X) * _carBody.AngularVelocity;
        return (_carBody.LinearVelocity + tangentialVelocity) / PixelsPerMeter;
    }
}
```

To cleanly separate the engine logic from `Car3`, you can create a **dedicated `Engine` class** – it doesn’t need to be a `Node`; a plain C# class works perfectly because the engine’s state is purely numeric. The car will own an instance of `Engine` and call its update method each physics tick, supplying the necessary inputs (throttle, gear ratio, rear wheel speed, delta time). The engine returns the current RPM and the net engine torque, which the car then passes through the transmission (gear ratio and final drive) to the wheels.

Here’s a step‑by‑step plan:

## 1. Create a new `Engine.cs` class

Move these **properties and constants** from `Car3` into the engine class:
- `MaxEngineTorque`
- `IdleRPM`
- `RedlineRPM`
- `RevLimitHoldTime` (constant)
- Private state: `EngineRPM`, `_revLimiterTimer`

Also move the **method** `GetEngineTorqueAtRPM`, because it only depends on the engine’s own properties and parameters. You can make it a private helper, exposed through the main update method.

Add a public `Update` method that:
- Takes `float dt`, `float throttle`, `float currentGearRatio`, `float averageRearWheelOmega`.
- Updates `EngineRPM` based on wheel speed (like the “MOTOR DYNAMICS” section in `Car3`).
- Clamps RPM, handles the rev limiter.
- Computes engine friction/combustion torque via `GetEngineTorqueAtRPM`.
- Returns the **net engine torque** (combustion − friction) and optionally the new RPM (or you can expose RPM as a property).

*Example skeleton*:

```csharp
public class Engine
{
    public float MaxEngineTorque { get; set; } = 233f;
    public float IdleRPM { get; set; } = 1000f;
    public float RedlineRPM { get; set; } = 7000f;
    private const float RevLimitHoldTime = 0.15f;

    public float EngineRPM { get; private set; } = 1000f;
    private float _revLimiterTimer = 0f;

    /// <returns>Net engine torque (Nm) available at the flywheel.</returns>
    public float Update(float dt, float throttle, float currentGearRatio, float averageRearWheelOmega)
    {
        // 1. Update RPM from wheel speed (neutral/in‑gear logic)
        if (currentGearRatio == 0f) // Neutral detection
        {
            float targetRPM = throttle > 0.1f ? RedlineRPM : IdleRPM;
            EngineRPM = Mathf.MoveToward(EngineRPM, targetRPM, 4000f * dt);
        }
        else
        {
            float calculated = Mathf.Abs(averageRearWheelOmega * Mathf.Abs(currentGearRatio) * _finalDriveRatio * (60f / (2f * Mathf.Pi)));
            EngineRPM = Mathf.Lerp(EngineRPM, calculated, 15f * dt);
            EngineRPM = Mathf.Clamp(EngineRPM, IdleRPM, RedlineRPM);
        }

        // 2. Rev limiter
        if (_revLimiterTimer > 0f)
            _revLimiterTimer -= dt;

        // 3. Compute torque using the same RPM‑dependent curve as before
        float engineTorque = GetEngineTorqueAtRPM(EngineRPM, throttle, dt);
        return engineTorque;
    }
```

- You can still expose engine‑tuning values in the car’s inspector by forwarding them. For example:

  ```csharp
  private float MaxEngineTorque { get => _engine.MaxEngineTorque; set => _engine.MaxEngineTorque = value; }
  ```

## 3. What to keep in `Car3` (transmission / gear logic)

Everything related to **gear selection** and **gear ratios** stays in the car (or could be moved to a separate `Transmission` class later). Specific items:

- `_gearRatios` array (gear ratios)
- `_currentGearIndex`
- `_finalDriveRatio`
- `UpdateGearInput()` and `GetGearName()`
- The code that splits `BrakeTorque` and calls `Wheel.UpdatePhysics`

These pieces are still the car’s responsibility because they tie directly to player input and wheel assignment.

## 4. Optional: Turn the engine into a `Resource`

If you want to be able to swap engines easily (e.g., drop a `SportsEngine.res` onto the car), you can make `Engine` inherit from `Resource` and export a field like:

```csharp
[Export] public Engine EngineSetup;
```

Then in `_Ready`, clone it or just use it directly. That gives you visual editor configuration. But for now a plain class is perfectly fine and simpler.

## Benefits

- **Clear separation of concerns** – The car no longer contains the RPM‑curve and rev‑limiter logic.
- **Testability** – You can unit‑test the engine’s torque curve independently of the physics.
- **Reusability** – If you build another car, you can reuse the same `Engine` class with different tuning values.

## One more thought: Neutral detection

Currently `Car3` checks `if (_currentGearIndex == 0)` to decide whether the engine is idling or spinning freely. In the engine’s `Update`, you need to know if the transmission is in neutral. The cleanest way is to **pass the actual gear ratio** (0 for neutral) instead of a separate flag. Then inside `Engine.Update` you can test `if (Mathf.Abs(gearRatio) < 0.001f)`. This keeps the engine unaware of gear indices.

That’s the whole process. Once you’ve extracted the engine, you can continue decoupling – e.g., pulling the gear management into a `Transmission` class, or even making the wheels scenes, as you asked earlier. The architecture stays flexible.
